Methodology & Scoring Criteria

This report presents the findings of an automated Dynamic Application Security Testing (DAST) scan. The scan involved sending various malicious payloads and unexpected inputs to the target API to identify security vulnerabilities.

Orthrus utilizes an advanced taxonomy to categorize findings.
  • CVSS 3.1 Base Score: Provides a theoretical numerical score (0.0 to 10.0) reflecting the intrinsic qualities of a vulnerability.
  • CAPEC: Common Attack Pattern Enumeration and Classification, identifying the specific attack mechanisms used.
  • OWASP: Maps the findings to the latest API Security Top 10 guidelines.
  • Global Grade: A letter grade (A to F) assessing the overall security posture based on the highest severity vulnerabilities found.
